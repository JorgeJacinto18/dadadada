Levantar un Servidor DNS

El sistema de nombres de dominio o DNS (Domain Name System) es un conjunto de protocolos y servicios que nació con la finalidad de evitar a las personas usuarias el esfuerzo de tener que recordar una dirección IP. Sin lugar a duda, nos resulta más fácil recordar un nombre, y además la dirección numérica podría cambiar por diferentes razones, sin que por ello cambie el nombre.

Pasos para hacerlo

Lo primero que debemos de hacer es convertinos en root, para ello necesitamos haber creado un usuario normal y un root, algo que la hicimos anteriormente. Una vez hecho, proseguimos con los siguientes pasos.

Primero debemos de instalar el servicio/paquete bind con el siguiente comando: apt-get install bind9

Nos moveremos/cambiamos de directorio para realizar la configuración del DNS con el comando: cd /etc/bind

Dentro del directorio, debemos modificar un archivo que contiene el direcotio llamada named.conf.local, para verificar que si esta, podemos usar el comando ls -l y despues usar vi named.conf.local para modificarlo.

Una vez dentro para mdificar el archivo, tendremos texto dentro que nos servida para configurar para nuestro DNS, con ello debermos de usar la siguiente estructura:

zone "midominio(ejemplo: acd.ejemplo.com)" {

type master;

file "ubicacion del archivo (ejemplo: /etc/bind/db.acd)"

};

Guardamos el archivo, regresando a la carpeta en donde el archivo se encuentra. Estando ahí,vamos a copiar un archivo que usaremos como base para editarlo a nuestras necesidades. El archivo se llama db.local, y para copiarlo usaremos el siguiente comando: cp db.local db.acd Nota: el nombre que pusiste en file va en acd

Ahora editaremos el archivo con VI: vi db.acd, dentro lo adaptaremos a nuestras necesidades

Que hacer dentro del archivo db.

El archivo posee una estructura, la cual mostraremos parcialmente a continuación y marcaremos en negritas las partes que deben ser modificadas. NOTA: En donde indique tudomino deberas de cambiarlo con el nombre de tu domino en ves de dejar el ejemplo usado

BIND data file for abc.ejemplo.com(tudominio)

@    IN    SOA     acd.ejemplo.com(tudominio)      root.acd.ejemplo.com(tudomino)
     IN  NS  ns1.acd.ejemplo.mx(tudominio)
ns1    IN  A   127.0.0.1(TudominioIPv4)
ns1    IN  AAAA   2001:0db8:85a3:0000:0000:8a2e:0370:7334(TudominioIPv6 si la posees)

Lo anterior seria lo más ensencial para levantar un servidor DNS, sin embargo mostraremos la estructura basica para hacer registro y el siginicado de algunos terminos usados en los registros

Estructura de los registros DNS

Nombre registro    Protocolo en uso    Tipo registros   Direccion o nombre

Ejemplo: paja IN A 148.202.48.1

Algunos terminos

$TTL: Tiempo de vida de los registros

@: Hace referencia al nombre de nuestro dominio

IN: Indica el protocolo(En este caso, de Internet)

SOA: Registro de autoridad

NS: Name Server o Nombre del servidor

A: Direccion de IPv4

AAAA: Direccion de IPv6
